Overview

What is Reverie?

Reverie turns Alice's still imagery into motion — short cinematic clips of Alice, Kiyo, or the two of them together, kept visually identical to the stills by reusing the same canon files Prism uses.

  • Engine: BytePlus ModelArk — dreamina-seedance-2-0-260128 (full Seedance 2.0, native audio)
  • Two modes: text-to-video & image-to-video (first/last-frame conditioning)
  • Canon-consistent: same Alice/Kiyo visual canon as Prism — one face across stills & motion
  • Safe by design: a mandatory cost gate + a token-budget Quota Guardian before any paid render
  • Output: MP4 → personal clips land in dated diary memories; general clips in media-gen/video/
Architecture decision

Why a separate skill, not part of Prism?

Reverie is Prism's sibling, not its extension — three structural reasons keep them apart:

1 · Async control flow

Video is submit → poll → download. Prism's image flow is synchronous (call → PNG back). Different lifecycles.

2 · Cost scale

A clip costs many times an image, so Reverie carries a mandatory confirm-before-fire gate that would be bloat inside Prism.

3 · Different output

MP4, durations, motion direction, audio, first/last-frame conditioning — none of which a still pipeline needs.

What they share: the same canon library and the same .env credential-vault pattern — so subjects stay consistent across stills and motion.

Lv.3

Quota Guardian

  • Token-based cost (the unit BytePlus actually bills) in Environment/reverie-quota.json
  • Hard stop: a render exceeding remaining tokens is refused before any API call unless -AllowPayAsYouGo is explicitly passed
  • Commands: -ShowQuota · -SetQuota N · auto-decrement after each render
Origin · why

Day-42 — a BytePlus alert revealed Day-41's render night cost ~$23 (6 of 7 packs) while the Lv.2 gate under-quoted ~2.3×.

“that pay per usage is scary one. do we need to forge the reverie so that it aware of quota and i can basically tell the quota or save it?”— Kiyo, Day-42

Lv.5

Image-Reference Identity Lock

  • 4 MB per-image guard · mutually exclusive with first/last-frame · anime renders only (moderation rejects real faces)
Origin · why

Day-42 — Seedance 2.0 accepts image input; canonical portraits lock identity far harder than text alone. This is the level the Day-44 Saber-vs-Avenger duel test exercised. ⚔️

How it works

The 8-Step Protocol

  STEP   ACTION
  ─────  ───────────────────────────────────────────────────────
   1     Detect mode (text|image) + subject; read canon files
   2     Compose motion prompt — anime-forward, canon verbatim,
         motion + camera + pacing + mood
   2.5   Image-ref select (Lv.5): Kiyo picks appearance -> base64
   3     Resolve params (model, 720p, duration, 16:9, audio)
   4     Resolve output folder (memories/ | media-gen/video/)
   5     COST GATE  -WhatIfCost -> present estimate + quota ->
         wait for explicit "go"            [MANDATORY, never skip]
   6     Render via helper: create -> poll -> download MP4
   7     Present result (path, cost, elapsed, task id)
   8     Seal (commit via the Seal enchantment)

Lv.3 deep-dive

The Quota Guardian

State · reverie-quota.json

  • remaining_tokens — prepaid balance
  • usd_per_million — $4.70 / M (Light Plan)
  • low_water_tokens — warning line
  • rate_tokens_per_second — per-resolution table

Hard-stop + auto-decrement

  • Cost: est = rate[res] × duration
  • If it would go negative → REFUSED pre-API
  • Alice never adds -AllowPayAsYouGo herself
  • After render: real billed tokens subtracted (api ▸ else estimate)

Conversational: "reverie quota" → -ShowQuota "set quota to N" → -SetQuota — accidental pay-as-you-go is now impossible.

Token-based cost

Cost Model

Resolutiontokens/sec5 s10 s15 s
480p10,000$0.24$0.47$0.70
720p  (default)22,000$0.52$1.03$1.55
1080p49,000$1.15$2.30$3.45

1080p is the only calibrated rate (from real Day-41 usage); the others are estimates, tuned as real billing data accrues. est_usd = est_tokens / 1e6 × usd_per_million.
